Cистеми Контролю Git
  • Централізовані VCS (CVCS): Одне центральне сховище, яке містить усі дані, а розробники підключаються до нього, щоб отримати або надіслати зміни. Приклад: Subversion (SVN).
  • Розподілені VCS (DVCS): Кожен розробник має повну копію історії змін, і зміни можуть синхронізуватися між різними копіями сховища. Приклад: Git, Mercurial.
    • Основні можливості Git
  • Гілки (branches): Можна створювати паралельні гілки для роботи над різними функціями або виправленнями багів, що дозволяє незалежно розвивати кілька напрямків проєкту.
  • Коміти (commits): Записи змін, які зберігають історію правок, кожен коміт містить опис змін і посилання на попередні версії.
  • Мердж (merge): Об'єднання гілок з різними змінами.
  • Система відкату: Можливість повертатися до будь-якої попередньої версії коду.
